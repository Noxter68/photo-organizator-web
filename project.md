# Project: Photo Organizer App Landing Page (SEO + Conversion Optimized)

## Objective

Create a high-converting landing page for a desktop application that automatically organizes photos using metadata.

Primary goals:

- Generate organic traffic from Google in French and English
- Convert visitors into buyers with a one-time payment model (50€ lifetime license)

Secondary goals:

- Clearly position the product as the easiest solution to organize iPhone / iPad / Mac photo exports on Windows
- Explain the product visually in a very simple way
- Make the buying and download process frictionless

---

## Core Value Proposition

Turn messy photo collections into clean, structured folders automatically.

Main angle:
When users move photos from iPhone, iPad or Mac to Windows, they often lose the album structure they created in the Apple ecosystem.

This product solves that pain point:

- It organizes photos automatically using metadata
- It recreates a clean folder structure on Windows
- It helps users recover order even if Apple albums are no longer preserved
- It is simple, local and fast

---

## Main SEO Positioning

The landing page must be optimized around a very specific and highly valuable search intent:

Users are not looking for “an innovative AI app”.
They are looking for:

- a way to organize photos automatically
- a way to sort iPhone photos on Windows
- a way to rebuild photo organization after leaving iOS / macOS

The page must therefore be built around problem/solution SEO.

## Payment Flow Requirements

I already have the product created on Stripe.

Implement:

- Stripe payment integration
- buy button connected to Stripe checkout flow
- payment success handling
- webhook verification on backend
- secure success page

Expected flow:

1. User clicks Buy now
2. User is redirected to Stripe Checkout
3. User completes payment
4. Stripe sends webhook to backend
5. Backend verifies the payment
6. Success page displays payment confirmation
7. Download button becomes available only after confirmed successful payment

Important:
Do not expose the download before payment is actually confirmed.

---

## Download Delivery Requirements

After a successful Stripe payment:

- Show a clear success state
- Display a download button for the Windows executable
- Optionally also send a confirmation email with download instructions
- Download access should be linked to confirmed payment status

The UX must be very simple:
Payment successful → click to download

---

## File Hosting Requirements

The Windows executable must not be bundled directly in the frontend app.

Recommended direction:

- host the executable in object storage
- generate a secure download link after successful payment
- keep the process simple and reliable

The implementation should support:

- easy file replacement when publishing a new version
- secure or temporary access to the downloadable file
- stable hosting for production use
